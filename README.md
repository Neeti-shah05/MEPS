This project analyzes patient-level data from the Medical Expenditure Panel Survey (MEPS) to predict cardiovascular outcomes and segment patient populations by health burden. The dataset contains 22,431 records and over 1,400 variables, narrowed down to 9 clinically relevant features: age, sex, race, region, insurance status, heart disease, angina, heart attack, and stroke.
The pipeline begins with data preparation, where raw MEPS column codes are renamed for readability, missing values are imputed using column medians, and a custom filtering function removes any records with non-informative survey responses such as "Inapplicable", "Refused", or "Don't Know" from the disease columns, keeping only clean binary Yes/No responses for modeling.
Exploratory data analysis includes a correlation heatmap revealing that age has strong positive correlations (above 0.6) with all four cardiovascular conditions, and that the disease indicators themselves are nearly perfectly correlated with each other, reflecting shared clinical risk profiles. Cross-tabulations of stroke diagnosis by insurance type and categorical count plots for all demographic and health variables round out the EDA.
Two logistic regression models are built using Scikit-learn. The first predicts stroke diagnosis from age and cardiovascular comorbidities. The dataset has a severe class imbalance of roughly 18:1 (no stroke vs stroke), so random oversampling via sklearn.utils.resample is applied to the training set before fitting. Despite achieving 95% overall accuracy, the model fails to detect stroke cases meaningfully, with an AUC of 0.20, illustrating the classic class imbalance trap where accuracy is a misleading metric. The second model predicts insurance coverage status (insured vs uninsured) using demographic and health features, achieving a more informative AUC of 0.69.
K-Means clustering is applied to segment patients by chronic disease burden and insurance status. Features are standardized using StandardScaler before clustering. Silhouette analysis across k values from 2 to 5 confirms k=3 as optimal with a score of 0.41, producing three interpretable patient segments corresponding to low, moderate, and high cardiovascular disease burden.
Reusable functions for label mapping, binary response filtering, and logistic regression training are factored into a separate Python module, keeping the notebook clean and the pipeline consistent across both models.
